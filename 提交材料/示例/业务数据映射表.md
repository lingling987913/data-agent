# ywdata / 评审材料 → 提交包示例映射表

> **数据源（优先级由高到低）**：
> 1. 仓库根 `ywdata/`（GNC 业务样例，默认 `.gitignore` 不入库）
> 2. **`提交材料/评审材料/`**（提交包内正式测试/评审材料，含月兔一号/蓬莱一号四件套）
>
> **提交包 测试数据**：
> - 已随包提供**脱敏最小样例**（`示例/01–04/测试数据/`，由 `示例/脚本/生成最小测试数据.py` 生成，可离线冒烟）；
> - 推荐执行 `bash 提交材料/示例/脚本/复制业务数据.sh`：有 `ywdata/` 时用 ywdata，否则自动从 **`评审材料/月兔一号`** 同步真实材料。
>
> **命名约定**：fixture 采用中文短名，README 中标注原始 评审材料/ywdata 路径与业务含义。
>
> **任务一范围**：任务一 **ywdata 实跑验证**覆盖 GNC 送审面向文档（PDF / Word / Excel 等）的可观测能力；**HTML/PPT、极端 OCR、工程图/流程图**不在本次验证范围（代码或 golden 合成 case 另见示例 5 / 任务三）。

## 不在本次任务一验证范围

| 项 | 说明 |
| --- | --- |
| HTML / PPT 解析 | 无 ywdata 样例；`html_parse` / `pptx_parse` 为**任务三**合成基准回归 |
| 极端 OCR | 模糊拍照、光线不均、手写、签章重叠等；CMG50 仅覆盖**常规** PDF |
| 工程图 / 流程图 | 无独立图件基准；正文内嵌简图不单独评测 |

## 离线基准与 ywdata 关系

| 机制 | 数据 | 用途 |
| --- | --- | --- |
| **离线基准门禁** | `tests/fixtures/review_docs/*.md` + 运行时生成 docx/xlsx/pptx/html/image | **任务三**回归；合成 GNC 示范审查对象；**不等同于任务一 ywdata 实跑** |
| **ywdata 在线演示** | `示例/01–03/测试数据/` | **任务一**主验证路径 |
| **二者关系** | 互补非等同 | 离线基准 F1 **不能**直接等同于 ywdata Word/PDF 实跑分数 |
