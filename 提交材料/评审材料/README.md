# 测试 / 评审材料（评审材料）

本目录为**组委会正式测试与评审用业务文档**，随提交包一并提供。示例 **01–04** 的 `测试数据/` 可由此目录同步为真实材料（见下方「与 示例 的关系」）。

## 目录结构

```
评审材料/
├── 月兔一号/    # 月兔一号飞轮文档包（竞赛题一，示例 01–04 主数据源）
│   ├── 月兔一号飞行器飞轮研制任务书.docx
│   ├── 月兔一号飞行器飞轮可靠性安全性设计与分析报告.docx   (~1.6 MB)
│   ├── 产品保证工作检查单（公开）.docx
│   └── 文档检查需求（公开）.xlsx
└── 蓬莱一号/    # 蓬莱一号飞轮文档包（结构同月兔一号，可用于跨型号扩展验证）
    ├── 蓬莱一号飞行器飞轮研制任务书 .docx
    ├── 蓬莱一号飞行器飞轮可靠性安全性设计与分析报告.docx
    ├── 蓬莱一号产品保证工作检查单（公开）.docx
    └── 文档检查需求（公开）.xlsx
```

> **说明**：`~$*.docx` 为 Word 临时锁文件，可忽略，不参与复制与测试。

## 与 提交材料 其他部分的关系

| 位置 | 角色 |
| --- | --- |
| **评审材料/**（本目录） | 正式测试/评审材料**真源**；中文原名，按月兔一号/蓬莱一号分题 |
| **示例/01–04/测试数据/** | 工程验证用 fixture；中文短名，由本目录或 `ywdata/` 同步 |
| **示例/** README | 各示例的 API curl、能力说明、输出样例 |
| **测试结果/** | golden / pytest **实跑输出**（与 评审材料 材料互补，见该目录 README） |
| **仓库根 ywdata/** | 与 评审材料 同结构的业务样例（默认 `.gitignore`）；有 ywdata 时优先于 评审材料 |

## 同步到 示例 测试数据

```bash
cd /path/to/data-agent

# 优先 ywdata/；无 ywdata 时自动回退 提交材料/评审材料/月兔一号
bash 提交材料/示例/脚本/复制业务数据.sh

# 或显式指定 评审材料
TOREVIEW=提交材料/评审材料 bash 提交材料/示例/脚本/复制业务数据.sh
```

同步后 fixture 英文名映射见 [示例/业务数据映射表.md](../示例/业务数据映射表.md)。
