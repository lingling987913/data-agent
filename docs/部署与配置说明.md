# 新人部署与配置说明

面向刚 clone 仓库的同事：用一条脚本完成依赖安装与前后端启动，并按本文配置环境变量。

## 1. 环境要求

| 组件 | 要求 | 安装参考 |
| --- | --- | --- |
| Python | ≥ 3.10 | 系统包或 pyenv |
| [uv](https://docs.astral.sh/uv/) | 推荐 | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| [Bun](https://bun.sh) | 前端必需 | `curl -fsSL https://bun.sh/install \| bash` |
| poppler-utils | 可选 | `pdftotext` 作 PDF 兜底：`apt install poppler-utils` / `brew install poppler` |

操作系统：Linux、macOS 或 WSL2 均可。Windows 建议使用 WSL2。

## 2. 一键部署

在仓库根目录执行：

```bash
chmod +x scripts/deploy.sh scripts/prod.sh scripts/dev.sh

# 首次：安装依赖 + 生成 .env + 生产模式启动前后端
./scripts/deploy.sh

# 仅安装依赖、复制环境文件，不启动
./scripts/deploy.sh --setup-only

# 开发模式（热更新前端、后端默认 8081）
./scripts/deploy.sh --dev

# 检查配置是否仍为占位符
./scripts/deploy.sh --check

# 停止
./scripts/deploy.sh -k
```

启动成功后：

| 服务 | 生产模式 (`deploy.sh`) | 开发模式 (`deploy.sh --dev`) |
| --- | --- | --- |
| 后端 API | http://127.0.0.1:8080/docs | http://127.0.0.1:8081/docs |
| 前端工作台 | http://127.0.0.1:3000/super-agent | 同上 |
| 日志 | `.prod/logs/` | `.dev/logs/` |

底层仍使用 `scripts/prod.sh` / `scripts/dev.sh`；日常重启可直接 `./scripts/prod.sh restart`。

## 3. 必须配置 vs 可选配置

### 3.1 必做（否则易踩坑）

| 文件 | 变量 | 说明 |
| --- | --- | --- |
| 根目录 `.env` | `API_TOKEN` | REST 认证密钥。内网开发可用默认 `dev-token-change-me`；**公网或多人共用机器必须改成强随机串** |
| `web/.env.local` | `NEXT_PUBLIC_API_TOKEN` | **必须与根目录 `API_TOKEN` 完全一致**，否则前端请求 401 |

首次运行 `./scripts/deploy.sh` 会从 `.env.example` / `web/.env.example` 复制上述文件（已存在则不会覆盖）。

### 3.2 强烈建议（完整审查能力）

| 文件 | 变量 | 说明 |
| --- | --- | --- |
| `.env` | `LLM_API_KEY` | 文本模型：审查、Super-Agent、编排、结构化等 |
| `.env` | `LLM_BASE_URL` | 默认 DashScope 兼容端点 |
| `.env` | `LLM_MODEL_NAME` | 如 `qwen3.6-27b` |
| `.env` | `VLM_API_KEY` | 视觉模型：嵌入图、图块描述（可与 LLM 同 Key） |
| `.env` | `VLM_BASE_URL` / `VLM_MODEL_NAME` | 须为支持 `image_url` 的多模态模型 |

未配置 LLM 时：健康检查、部分规则步骤、离线 Golden 仍可跑；**LLM Agent 审查步骤会降级或跳过**。

### 3.3 PDF / 扫描件解析（按需）

| 场景 | 配置 |
| --- | --- |
| 本机已部署 MinerU HTTP 服务 | `MINERU_LOCAL_ENABLED=1`、`MINERU_LOCAL_FIRST=1`、`MINERU_LOCAL_API_BASE=http://127.0.0.1:8000`（端口以实际为准） |
| 使用 mineru.net 在线 API | `MINERU_EXTRACT_API_ENABLED=1`、填写 `MINERU_AGENT_API_TOKEN`（v4 extract） |
| 评审机 / 无 MinerU | `MINERU_LOCAL_ENABLED=0`，关闭在线 API 时可设 `MINERU_EXTRACT_API_ENABLED=0`，走 `pdftotext` 兜底 |

开发脚本默认 `MINERU_LOCAL_ENABLED=0` 以加快启动；生产 `prod.sh` 同样默认关闭，以 `.env` 为准。

### 3.4 公网或竞赛暴露（按需）

| 变量 | 说明 |
| --- | --- |
| `API_EXPOSE_SCOPE` | `full`（本地默认）/ `competition`（仅健康检查 + 竞赛三端点）/ `demo` |
| `INTERNAL_API_TOKEN` | 内网旁路，配合请求头 `X-Internal-Token`（勿写入 `NEXT_PUBLIC_*`） |
| `CORS_ORIGINS` | 前后端不同域时补充前端 origin |

### 3.5 前端专项（`web/.env.local`）

| 变量 | 说明 |
| --- | --- |
| `NEXT_PUBLIC_COMPREHENSIVE_REVIEW_MINERU_PARSER` | 综合审查页解析器，默认 `mineru` |
| `NEXT_PUBLIC_COMPREHENSIVE_REVIEW_MINERU_PARSE_MODE` | 如 `auto` / `ocr` / `txt` |

构建生产前端时，`DATA_AGENT_API_ORIGIN` 会写入 Next.js 的 `/api` 代理目标（默认 `http://127.0.0.1:8080`）。改端口后需重新 `./scripts/prod.sh build` 或 `./scripts/deploy.sh`（会自动检测并重建）。

### 3.6 可选增强

| 变量 | 说明 |
| --- | --- |
| `LIGHT_LLM_*` / `LIGHT_VLM_*` | 轻量模型，解析/公式场景优先 |
| `REVIEW_PLUS_AGENTS_ENABLED=1` | 启用 Review-Plus 中 LLM Agent 步骤 |
| `PARSING_ENABLE_IMAGE_DESC=1` | Word 嵌入图 + VLM 描述 |
| `RAGFLOW_*` | GNC 材料同步到 RAGFlow（可选） |

完整变量注释见仓库根目录 [`.env.example`](../.env.example)。

## 4. 最小可运行示例

**仅验证安装、健康检查、离线基准**（可不配 LLM / MinerU）：

```bash
./scripts/deploy.sh --setup-only
# 编辑 .env：保持 API_TOKEN=dev-token-change-me，并设置：
#   MINERU_LOCAL_ENABLED=0
#   MINERU_AGENT_API_ENABLED=0
#   MINERU_EXTRACT_API_ENABLED=0
./scripts/deploy.sh
curl -s http://127.0.0.1:8080/health
```

**内网完整工作台**（LLM + 本地 MinerU）：

```bash
# .env 片段
API_TOKEN=your-strong-token-here
LLM_API_KEY=sk-...
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen3.6-27b
VLM_API_KEY=sk-...
MINERU_LOCAL_ENABLED=1
MINERU_LOCAL_API_BASE=http://127.0.0.1:8000

# web/.env.local
NEXT_PUBLIC_API_TOKEN=your-strong-token-here
```

## 6. 常见问题

| 现象 | 处理 |
| --- | --- |
| 401 Invalid API token | 核对 `API_TOKEN` 与 `NEXT_PUBLIC_API_TOKEN` |
| 前端连不上 API | 生产模式确认后端在 8080；开发模式 dev 用 8081，且 `DATA_AGENT_API_ORIGIN` 由 `dev.sh` 注入 |
| `未找到 .venv` | 先执行 `./scripts/deploy.sh --setup-only`（内部 `uv sync`） |
| MinerU health degraded | 本地服务未起或改 `MINERU_LOCAL_ENABLED=0` 接受降级 |
| 竞赛 result 409 | 任务未完成，先查 `/api/v1/task/status/{id}` |

更细的评审/竞赛说明见 [提交材料/部署与运行说明.md](../提交材料/部署与运行说明.md)。

## 7. 脚本对照

| 脚本 | 用途 |
| --- | --- |
| `scripts/deploy.sh` | **新人推荐**：依赖安装 + env 初始化 + 调用 prod/dev |
| `scripts/prod.sh` | 生产：构建 Next standalone + 后台启停 |
| `scripts/dev.sh` | 开发：uvicorn（可选 reload）+ `bun run dev` |
